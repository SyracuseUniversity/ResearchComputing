üìñ What's Different About Research Computing?

Coming from a laptop or desktop? Research clusters work differently:

  • Batch computing: Write scripts and submit jobs to a queue (not interactive)
  • Shared resources: Hundreds of researchers share the same hardware
  • Command-line only: No GUI tools or IDEs on clusters
  • Software management: Use conda, containers, or modules (can't install directly)

This enables fair resource sharing, running jobs too large for any single computer, and accessing specialized hardware like GPUs.

Cluster Specifications

Technical details for our HTC and HPC clusters.

OrangeGrid (HTCondor) - High-Throughput Computing

Cluster Type: High-throughput computing cluster with over 80,000 cores

Best For:

  • Parameter sweeps and batch processing
  • Many independent jobs (hundreds to thousands)
  • Embarrassingly parallel workloads
  • Single-node high-memory or GPU jobs
  • LLM inference across multiple inputs

Resources:

  • Over 80,000 cores (AMD and Intel processors)
  • Heterogeneous pool with varying configurations
  • High-memory nodes available
  • GPU nodes available (via SUrge infrastructure)

Available GPUs: NVIDIA A100, L40S, A6000 (via SUrge)
Request with: +request_gpus = 1

Storage: 4TB home directory (NetApp, mounted on all nodes)
Access: SSH via its-og-loginX.syr.edu

üìä Detailed specifications: OrangeGrid Technical Details

Zest (Slurm) - High-Performance Computing

Cluster Type: High-performance computing cluster with over 25,000 cores and InfiniBand interconnect

Best For:

  • Multi-node parallel jobs (MPI)
  • Tightly-coupled computations requiring fast interconnect
  • Large-scale parallel workloads (hundreds of cores)
  • GPU-accelerated research
  • Long-running jobs (up to 40 days)
  • High-memory workloads

Resources:

  • Over 25,000 cores interconnected with InfiniBand
  • Multiple partitions optimized for different workloads
  • Extended runtime options (longjobs partition: 40 days)

Available GPUs: NVIDIA A40 (primary, via SUrge)
Request with: #SBATCH --gres=gpu:1

Storage: 4TB home directory (NetApp, mounted on all nodes)
Access: SSH via its-zest-loginX.syr.edu

Common Tasks

Quick reference for everyday cluster operations.

Job Submission

Task OrangeGrid (HTCondor) Zest (Slurm)
Submit job condor_submit job.sub sbatch script.sh
Check status condor_q netid squeue -u netid
Cancel job condor_rm jobid scancel jobid
View resources condor_status sinfo
Job history condor_history netid sacct

Software Management

Create Conda Environment

# Load anaconda module (Zest only)
module load anaconda3

# Create environment
conda create -n myenv python=3.11

# Activate
conda activate myenv

# Install packages
conda install numpy pandas
pip install your-package

GPU Jobs

OrangeGrid: Add to submit file:

+request_gpus = 1

Zest: Add to SBATCH script:

#SBATCH --partition=gpu_zone2,gpu
#SBATCH --gres=gpu:1

File Transfer

# Upload to cluster
scp file.txt netid@cluster:~/path/

# Download from cluster
scp netid@cluster:~/results.txt ./

# Large transfers (resumable)
rsync -avz --progress /local/data/ netid@cluster:~/data/

For large data transfers (TBs): Contact researchcomputing@syr.edu for optimized transfer options.

Load Software Modules (Zest Only)

# See available modules
module avail

# Load module
module load anaconda3

# List loaded modules
module list
